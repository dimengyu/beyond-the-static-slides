5 Modes of Research Communication

The Premise

One Dataset, Many Visual Languages

Academic Peer Review

Demands reproducible, precisely encoded figures. Static is not a limitation — it is the requirement for scientific rigor and publication standards.

Clinical Operations

Demands filterable, comparative views that support repeated use. Interactivity enables the exploratory patterns of teams monitoring populations over time.

Point-of-Care Encounter

Demands immediate legibility on a small screen with minimal cognitive load. The design space narrows to what can be read and acted upon in seconds.

The same underlying data, encoded differently for each context. No format is superior — each is optimal for its audience, task, and environment.

Static Figures

Fixed-Format Scientific Encoding

Print-optimized figures where every design decision — color, scale, annotation, layout — is fixed at render time. The absence of interactivity is intentional: reproducibility demands that every reader sees exactly the same visual.

Multi-Panel Figure Architecture Static

Composition as Argument

Shared Axis Alignment

Panels that share a common x- or y-axis let readers compare distributions directly without re-reading scale labels — the visual grammar of the layout encodes the comparison structure.

Faceting by Variable

Splitting by subgroup (by race, center, time period) across panels converts a single cluttered chart into a small-multiple array — each panel readable in isolation, the pattern readable across the whole.

Annotation as Evidence

Significance brackets, confidence bands, and at-risk tables embedded within the figure deliver statistical context without requiring the reader to cross-reference a table.

Design Principles

A

Alignment

Grid-lock panels to a shared baseline so the eye moves horizontally, not hunting for axes.

R

Redundancy Reduction

One shared legend, one shared title — no repeated labels. Every ink mark earns its place.

H

Hierarchy

Panel order mirrors narrative order — the reader's eye should travel through the figure the same way it travels through the argument.

The Wall of Static Static

Where Static Breaks Down

Fixed Viewport, Hidden Data

Overplotting, overlapping labels, and compressed axes are permanent. A static figure cannot respond to a reader's need to zoom in, filter, or isolate a subgroup.

One State, One Moment

A published figure is frozen at the time of render. Updated data requires a new figure — there is no mechanism to refresh without re-exporting and republishing.

No Reader Agency

The author decides what is visible. Readers who want to compare a specific subgroup, re-sort a ranking, or examine an outlier are left with the author's single chosen view.

When to go beyond static

Data has more than ~3 dimensions that the reader needs to explore simultaneously

The audience needs to filter, drill down, or answer their own questions — not just receive yours

The underlying data updates regularly and the figure must stay current without manual re-export

Static remains the right choice for peer review, print, and any context demanding exact reproducibility

Interactive Charts

Browser-Rendered, User-Controlled Views

The data structure is fixed, but the visual state is user-controlled. Filtering, zooming, and hovering shift interpretive authority from author to reader — the same dataset surfaces differently for every person who explores it.

From Exploration to Operations

The Interactive Figure's Ceiling

One Question at a Time

Each chart is built around a single analytical frame. Exploring a second dimension — a different cohort, a different time window — requires navigating away and starting over.

Ephemeral State

Filtered views and zoom levels live only in the session. There is no persistent URL encoding the exact subgroup a clinician found useful yesterday — every visit starts blank.

No Shared Operational View

A scatter plot cannot simultaneously show a trend, a distribution, and a KPI summary. The team navigates between figures rather than reading a unified picture.

What dashboards add

Multiple panels, one viewport — KPIs, trends, and breakdowns coexist in a single screen, eliminating the need to navigate between charts

Persistent, bookmarkable state — the same URL returns the same filtered view, making dashboards a reusable operational artifact rather than a one-time exploration

Designed for repeated use — navigated rather than read, visited weekly rather than shared once; the interface assumes a user who returns, not a reader who arrives once

From the Office to the Bedside

Where Dashboards Can't Follow

Requires a Seated Workflow

Dashboards assume a user at a workstation. Clinical encounters happen at bedsides, in exam rooms, and in hallways — contexts where opening a browser and navigating a multi-panel layout simply isn't feasible.

Screen Size Mismatch

The information density that makes a dashboard powerful on a 27-inch monitor makes it illegible on a phone. Multi-panel grid layouts don't collapse gracefully to a 6-inch screen.

Login & Load Friction

Authenticating, loading a JavaScript-heavy report, and maintaining a session creates friction that breaks the pace of a brief clinical encounter — the window for decision support closes in seconds.

What mobile accessibility unlocks

Always in pocket — a native app requires no browser, no login at the point of care; it opens in one tap on a device clinicians and patients already carry

Designed for 5-second interactions — touch-optimized inputs, minimal cognitive load, and a single output screen match the rhythm of a brief clinical encounter

Personalized to the patient present — inputs collected in real time feed a model for this patient, this moment — not a population average displayed on a shared monitor
